# Audit: Audience, Placements, and Trends Tabs Show No Data

## Root Cause Analysis

**It's primarily a code-level issue, not a config or API-return issue.** There are three distinct problems:

---

### Problem 1 (ROOT CAUSE): Insights data is fetched on-demand per ad, but the `id` parameter may not match

**The flow for each tab:**
1. Frontend calls `useMetaAdInsightsPlacements(id)` etc., passing the ad's local DB ID
2. The hook calls the PHP backend `fetchMetaAdInsightsPlacements({ id })`
3. Backend runs `resolveAdMetaId($id)` — a SQL query joining `meta_ads` and `meta_ad_accounts`
4. If the ad is found, it hits the Meta Graph API `/{metaAdId}/insights` endpoint
5. Results are cached in `meta_ads_insights_cache` for 6 hours

**The problem:** There is no pre-syncing of per-ad insights data (demographics, placements, devices, daily trends). The daily sync (`syncAccountDailyInsights` at line 535) only populates `meta_ads_insights_daily` — used by the ad list date-range filter and the marketing dashboard. It does NOT populate `meta_ads_insights_cache`, which is what the Audience/Placements/Trends tabs read from.

This means every tab visit triggers **4 separate live Meta API calls** (daily + demographics + placements + devices). If any call fails (expired token, rate limit, permissions, ad not yet having delivery data), the result is an empty array.

**Key files:**
- `backend/src/MetaAdsApi.php:1564-1693` — the four `fetchMetaAdInsights*` methods
- `backend/src/MetaAdsApi.php:1533-1543` — `getCachedInsights()` returns cached data or null
- `backend/src/MetaAdsApi.php:535-607` — `syncAccountDailyInsights()` only populates `meta_ads_insights_daily`, NOT `meta_ads_insights_cache`

---

### Problem 2 (BUG): Empty results are cached for 6 hours, blocking retries

In `MetaAdsApi.php:1564-1596` (and the other three methods), when the Meta API returns an empty result set, the empty `[]` is still saved to cache:

```php
$data = array_map(function(array $r) { ... }, $rows);  // $rows could be []
$this->saveInsightsCache($id, 'daily', $data);          // caches [] for 6 hours
```

The `getCachedInsights()` method treats `[]` as valid cached data. So if the API temporarily returns nothing (ad paused, no delivery data yet, transient error), the empty result is locked for 6 hours with no retry path.

**Key file:** `backend/src/MetaAdsApi.php:1540-1541` — `is_array($decoded) ? $decoded : null` treats `[]` as valid

---

### Problem 3 (BUG): Missing `formatMetaAdsCurrency` import in Trends tab

`MetaAds.tsx` uses `formatMetaAdsCurrency` on lines 841 and 862 (inside the Trends tab Tooltip formatters), but it is never imported. The function is defined in `src/utils/metaAdsCurrency.ts`. The import is missing from `MetaAds.tsx:1-15`.

This will cause a `ReferenceError` at runtime when a user hovers over a tooltip in the Trends tab. The chart lines may still render (the formatter is lazy/deferred), but the tooltip interaction is broken.

**Key file:** `pages/MetaAds.tsx:841,862` (usage), `src/utils/metaAdsCurrency.ts:28` (definition)

---

### Problem 4 (UX): Errors are silently swallowed

All four insight methods catch exceptions and return `['data' => [], 'error' => ...]`. The frontend (`MetaAds.tsx:461-464`) only reads `.data` and ignores `.error`. Users see "No data available" with no indication of why.

---

## Recommended Fixes

### Fix 1: Do not cache empty results
In each of the four `fetchMetaAdInsights*` methods, only call `saveInsightsCache()` when `$data` is non-empty. This allows retries on the next page load.

**Files:** `backend/src/MetaAdsApi.php` — lines 1591, 1624, 1656, 1688

### Fix 2: Add missing `formatMetaAdsCurrency` import
Add to `MetaAds.tsx`:
```ts
import { formatMetaAdsCurrency } from '../src/utils/metaAdsCurrency';
```

**File:** `pages/MetaAds.tsx` (near line 10)

### Fix 3: Surface errors to the frontend
- Backend: the error field is already returned
- Frontend: read `dailyData?.error`, `demographicsData?.error` etc. and display a toast or inline error message when data is empty AND an error exists

**Files:** `pages/MetaAds.tsx:461-464`

### Fix 4 (optional, bigger): Pre-populate insights cache during sync
Modify `syncConnection()` or `syncAccountDailyInsights()` to also call the four insight endpoints per ad and populate `meta_ads_insights_cache`. This eliminates the 4 on-demand API calls per ad detail visit.

**File:** `backend/src/MetaAdsApi.php:535-607`

---

## Verification

1. After Fix 1: Clear `meta_ads_insights_cache` table, open an ad detail page, verify that empty results are NOT cached (check the table after page load)
2. After Fix 2: Open an ad detail page → Trends tab → hover over chart points → verify tooltip renders without JS error
3. After Fix 3: Simulate an API error (e.g., revoke token), open ad detail → verify the error message is shown instead of just "No data available"
4. After Fix 4: Run sync, then open ad detail → verify all four tabs show data without needing to hit the Meta API on-demand
